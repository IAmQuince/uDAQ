## In scope
- reconnect strategy selection after a real mid-run disconnect
- app-level lifecycle evidence for lost -> recovering -> recovered -> stabilized
- reconnect attempt traceability needed to prove which strategy ran first
- bounded regression coverage and package closeout

## Out of scope
- new hardware families
- UI redesign
- historian redesign
- hardware-permutation testing
- LabJack-only policy work that does not generalize to the main app lifecycle seam
