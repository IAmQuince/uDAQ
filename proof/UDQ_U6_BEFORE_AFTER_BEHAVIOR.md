# UDQ U6 Before/After Behavior

## Before Sprint 2
- the bounded real-U6 slice existed and worked for the proof path
- diagnostic truth around startup/runtime loss was relatively coarse
- hardware evidence return still leaned on console reading and copy/paste

## After Sprint 2
- the adapter exposes a structured status snapshot
- startup and recovery classifications are explicit
- bounded runtime degradation and disconnection counters are preserved in diagnostics
- a hardware run can be returned as a compact evidence bundle
